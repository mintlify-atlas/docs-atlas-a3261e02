# Installation

This guide provides comprehensive instructions for installing and setting up David Carrascosa's portfolio on your local development environment.

## Prerequisites

Before installing the portfolio, ensure your system meets these requirements:

### Required Software

<Steps>
  <Step title="Node.js 18 or Higher">
    The portfolio requires Node.js 18+ for modern JavaScript features and optimal performance.
    
    **Check your Node.js version:**
    ```bash
    node --version
    # Should output v18.0.0 or higher
    ```
    
    **Install Node.js:**
    - Download from [nodejs.org](https://nodejs.org/)
    - Or use a version manager like [nvm](https://github.com/nvm-sh/nvm):
    
    ```bash
    # Using nvm (recommended)
    nvm install 20
    nvm use 20
    ```
    
    <Warning>
      Node.js versions below 18 are not supported and may cause compatibility issues.
    </Warning>
  </Step>
  
  <Step title="Package Manager">
    Choose one of the following package managers:
    
    <Tabs>
      <Tab title="Bun (Recommended)">
        **Bun** is a fast JavaScript runtime and package manager. This project was built with Bun.
        
        ```bash
        # Install Bun (macOS/Linux)
        curl -fsSL https://bun.sh/install | bash
        
        # Verify installation
        bun --version
        ```
        
        [Bun documentation](https://bun.sh/docs)
      </Tab>
      
      <Tab title="npm">
        **npm** comes pre-installed with Node.js.
        
        ```bash
        # Verify npm installation
        npm --version
        
        # Update to latest version (optional)
        npm install -g npm@latest
        ```
      </Tab>
      
      <Tab title="yarn">
        **Yarn** is a popular alternative to npm.
        
        ```bash
        # Install Yarn globally
        npm install -g yarn
        
        # Verify installation
        yarn --version
        ```
      </Tab>
      
      <Tab title="pnpm">
        **pnpm** is a fast, disk-efficient package manager.
        
        ```bash
        # Install pnpm
        npm install -g pnpm
        
        # Verify installation
        pnpm --version
        ```
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Git">
    Git is required to clone the repository.
    
    ```bash
    # Check Git installation
    git --version
    ```
    
    **Install Git:**
    - macOS: `brew install git` or download from [git-scm.com](https://git-scm.com/)
    - Linux: `sudo apt install git` (Ubuntu/Debian) or `sudo yum install git` (CentOS/RHEL)
    - Windows: Download from [git-scm.com](https://git-scm.com/)
  </Step>
</Steps>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the portfolio repository from GitHub:
    
    ```bash
    git clone https://github.com/tu-usuario/web-portfolio.git
    ```
    
    <Note>
      Replace `tu-usuario` with the actual GitHub username or organization.
    </Note>
    
    Navigate to the project directory:
    
    ```bash
    cd web-portfolio
    ```
  </Step>
  
  <Step title="Install Dependencies">
    Install all required packages listed in `package.json`:
    
    <CodeGroup>
      ```bash bun
      bun install
      ```
      
      ```bash npm
      npm install
      ```
      
      ```bash yarn
      yarn install
      ```
      
      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
    
    This will install:
    - **Core dependencies**: React, React DOM, React Router
    - **Build tools**: Vite, TypeScript, SWC
    - **UI framework**: Tailwind CSS, shadcn/ui, Radix UI
    - **Animation**: Framer Motion
    - **Dev dependencies**: ESLint, Vitest, Testing Library
    
    **Installation time:** ~30-60 seconds (varies by internet speed)
    
    <Tip>
      The project includes a `bun.lockb` file for Bun users. If using npm, yarn, or pnpm, a new lockfile will be generated.
    </Tip>
  </Step>
  
  <Step title="Verify Installation">
    Verify that all dependencies were installed correctly:
    
    ```bash
    # Check that node_modules exists
    ls node_modules
    
    # Verify React is installed
    ls node_modules/react
    ```
    
    You should see a `node_modules` directory with all installed packages.
  </Step>
  
  <Step title="Start Development Server">
    Launch the Vite development server:
    
    <CodeGroup>
      ```bash bun
      bun run dev
      ```
      
      ```bash npm
      npm run dev
      ```
      
      ```bash yarn
      yarn dev
      ```
      
      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>
    
    **Expected output:**
    ```bash
    VITE v5.4.19  ready in 324 ms
    
    ➜  Local:   http://localhost:5173/
    ➜  Network: http://192.168.1.100:5173/
    ➜  press h + enter to show help
    ```
    
    <Note>
      The server listens on all network interfaces (`::`) by default, configured in `vite.config.ts:8`.
    </Note>
  </Step>
  
  <Step title="Access the Application">
    Open your browser and navigate to:
    
    ```
    http://localhost:5173
    ```
    
    You should see the portfolio homepage with:
    - Navigation bar
    - Hero section
    - About section
    - Experience timeline
    - Skills showcase
    - Contact form
    
    <Tip>
      Try resizing your browser window to see the responsive design in action.
    </Tip>
  </Step>
</Steps>

## Environment Configuration

This portfolio does not require environment variables for basic operation. However, if you plan to add features like:

- Contact form email delivery
- Analytics integration
- CMS integration

You may need to create a `.env` file:

```bash .env
# Example environment variables (not required for basic setup)
VITE_API_URL=https://api.example.com
VITE_CONTACT_EMAIL=contact@example.com
```

<Warning>
  Never commit `.env` files to version control. The `.gitignore` file already excludes them.
</Warning>

## Troubleshooting

### Common Issues

<Accordion>
  <AccordionItem title="Port 5173 Already in Use">
    **Error:** `Error: listen EADDRINUSE: address already in use :::5173`
    
    **Solution:** Use a different port:
    
    ```bash
    bun run dev --port 3000
    ```
    
    Or kill the process using port 5173:
    
    ```bash
    # macOS/Linux
    lsof -ti:5173 | xargs kill -9
    
    # Windows
    netstat -ano | findstr :5173
    taskkill /PID <PID> /F
    ```
  </AccordionItem>
  
  <AccordionItem title="Module Not Found Errors">
    **Error:** `Cannot find module '@/components/...'`
    
    **Solution:** Path aliases are configured in `vite.config.ts:16-18`. Ensure dependencies are installed:
    
    ```bash
    rm -rf node_modules
    bun install
    ```
    
    If using VS Code, reload the window: `Cmd+Shift+P` → "Reload Window"
  </AccordionItem>
  
  <AccordionItem title="TypeScript Errors">
    **Error:** Type errors in `.tsx` files
    
    **Solution:** Ensure TypeScript is installed:
    
    ```bash
    bun add -D typescript
    ```
    
    Restart your IDE's TypeScript server:
    - VS Code: `Cmd+Shift+P` → "TypeScript: Restart TS Server"
  </AccordionItem>
  
  <AccordionItem title="Tailwind CSS Not Working">
    **Error:** Styles not applying
    
    **Solution:** Verify Tailwind is configured:
    
    1. Check `tailwind.config.ts` exists
    2. Ensure `src/index.css` imports Tailwind directives:
    
    ```css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    ```
    
    3. Restart the dev server
  </AccordionItem>
  
  <AccordionItem title="Slow Build Performance">
    **Issue:** Vite is slow or unresponsive
    
    **Solutions:**
    
    1. **Clear Vite cache:**
    ```bash
    rm -rf node_modules/.vite
    ```
    
    2. **Exclude node_modules from IDE indexing** (VS Code settings):
    ```json
    "files.watcherExclude": {
      "**/node_modules/**": true
    }
    ```
    
    3. **Use SWC instead of Babel** (already configured in `vite.config.ts:14`)
  </AccordionItem>
  
  <AccordionItem title="Dependency Installation Fails">
    **Error:** Installation errors or network issues
    
    **Solutions:**
    
    1. **Clear package manager cache:**
    
    <CodeGroup>
      ```bash bun
      bun pm cache rm
      ```
      
      ```bash npm
      npm cache clean --force
      ```
      
      ```bash yarn
      yarn cache clean
      ```
    </CodeGroup>
    
    2. **Delete lockfile and reinstall:**
    ```bash
    rm bun.lockb  # or package-lock.json, yarn.lock
    rm -rf node_modules
    bun install
    ```
    
    3. **Use a different package manager** if one consistently fails
  </AccordionItem>
</Accordion>
